1. Introduction & Overview

Welcome to the Link Budget Simulator (Version 1.2.0) User Manual. This application is a web-based, baseband channel capacity simulation dashboard designed for hardware design and transmission engineering validation.

The simulator is designed to evaluate high-speed automotive communication standards over physical cable models, specifically adhering to guidelines within:

  • IEEE 802.3ch (Multi-Gigabit Automotive Copper Physical Layer Solutions - e.g., 2.5 Gbps, 5 Gbps, and 10 Gbps).
  • IEEE 802.3cy (Greater than 10 Gbps Electrical Alternative Physical Layer - e.g., 25 Gbps, 50 Gbps, and 100 Gbps Solutions over Coaxial and Shielded Differential Pairs).

The core simulator models signal attenuation, PCB trace degradation, transmit power masks, connector profiles, wire and reflection echoes, echo cancellation, AFE noise, and Forward Error Correction (FEC) algorithms to verify the overall **SNR Margin** of a link.

2. Technical & Math Core

The simulation engine evaluates the channel capacity by dividing the evaluation grid frequency band (from 0 up to Max Frequency) into discrete linear steps of size fStepHz = fMaxHz / nSteps. At each frequency step, it performs the following transmission line calculations:

2.1 Channel Insertion Loss Model (IL)

The cable insertion loss per meter is evaluated using physical fit constants ($b_0$, $b_1$, $p$, and the temperature coefficient $drho$):

IL_per_meter(f) = b0 * [(1 + drho * (T - 20)) * f]^p + b1 * f

The total loss at any step frequency includes the cable attenuation ($IL_{cable} \times CableLength$) plus the PCB trace attenuation ($IL_{pcb} \times PCBLength$).

2.2 Echo Contributions & Noise Model

The simulator models echo interference from connector interfaces and reflections along the wire:

  • Connector Echo Profile: Modeled via connector constant $C_0$ scaled by inline connector count and step frequency.
  • Residual Echo: Residual echo values are computed after applying Upstream and Downstream connector and reflection echo cancellation bounds (e.g. connectorEchoCancellationDb and cableReflectionEchoCancellationDb).
  • Noise Floor: Total linear noise at each step is the sum of AFE thermal noise density and residual echo noise power.

2.3 Slicer SNR Margin

The theoretical Slicer SNR is the weighted in-band average SNR calculated across the evaluated Nyquist band:

SNR_Theoretical = Integral_from_0_to_Nyquist( SNR(f) * df ) / NyquistFrequency

The Estimated Slicer SNR is then computed as the theoretical SNR minus the specified transceiver Implementation Loss. The **SNR Margin** displays the excess SNR available over the Required Slicer SNR (which is determined by target BER, PAM level, and FEC coding capability). A positive margin (margin ≥ 0 dB) results in a **PASS**, while a negative margin results in a **FAIL**.

4. Dashboard Indicators

The simulator outputs results across several high-visibility dashboards:

4.1 Summary Metric Cards

Positioned at the top of the results panel, these cards provide immediate visibility:

  • SNR Margin (dB): Displays the computed margin. Highlighted with a violet background for **Upstream** and a cyan background for **Downstream**. A **PASS** (green badge) indicates sufficient margins, while **FAIL** (red badge) indicates degradation below the BER target.
  • Nyquist (GHz): Displays the Nyquist frequency limit of the signal.
  • Est / Req (dB): Shows the side-by-side comparison of the Estimated Slicer SNR vs. the Required Slicer SNR.

4.2 Detailed Budget Tab

Shows an exact comparison grid of the parameters, detailing intermediate variables such as:

  • Baud Rate (GBaud): The actual line rate in gigasymbols per second.
  • FEC correction capability (t): Calculated error correction symbols, computed as t = Math.floor(((N - K) / 2) * efficiency).
  • Theoretical vs. Estimated Slicer SNR: Displays the influence of transceiver Implementation Loss.
  • Cable & PCB Loss at Nyquist: Breakdowns of attenuation levels measured exactly at the channel's Nyquist frequency.

5. Waveform Charts

The **Charts & Profiles** tab renders high-resolution interactive plots powered by Chart.js.

5.1 Chart Variants

  1. SNR Profile: Plots the Signal-to-Noise Ratio (in dB) across the frequency sweep.
  2. Insertion Loss Profiles: Displays three curves: Cable Insertion Loss, PCB Insertion Loss, and Total Channel Insertion Loss.
  3. Transmit PSD: Visualizes the power spectral density mask profiles (in dBm/Hz) for the upstream and downstream signal paths.

Y-Axis Span Capping Rule

To prevent extreme attenuation values (e.g. at high frequencies or long cables) from compressing the visible region of the chart, the simulator enforces a **maximum Y-axis range span of 150 dB**. If the data span exceeds 150 dB, the minimum display value is capped at max - 150, keeping peak profiles fully visible.

Linear X-Axis Tick Marks

The frequency x-axis runs on a continuous linear scale (GHz). Chart.js automatically computes nice, rounded numbers (like 1, 2, 3... GHz) for the axes, preventing cluttered tick labels with excess digits. Hovering over a chart displays precise tooltips formatted to 3 decimal places.

6. Advanced Functions

6.1 Backup & Config JSON

The **JSON Config** tab allows developers to inspect, export, or restore simulator states:

  • Save Config: Downloads a .json file containing the current input parameters values.
  • Load Config: Opens the browser's file examiner to upload a previously saved .json configuration. The simulator will automatically parse the file, populate the inputs fields, and run a simulation recalculation.
  • Copy to Clipboard: Copies the full JSON summary results (including calculated Baud, margins, and intermediate values) for integration into external data pipelines.

6.2 Print-Ready Simulation Reports

Clicking the **Generate Report** button (positioned next to the version badge in the navigation header) performs the following automated steps:

  1. Offscreen Compilation: Instantiates a temporary, hidden Chart.js context to draw SNR, Insertion Loss, and Transmit PSD profiles in a high-contrast, light-themed format suitable for printing.
  2. Waveform Extraction: Extracts base64 image strings from the offscreen canvases.
  3. Standalone Document Assembly: Compiles a print-friendly HTML document incorporating the date metadata, standard introductory notices, configuration summaries, performance grids, PASS/FAIL cards, and the embedded chart images.
  4. Download Trigger: Initiates a browser download for the compiled standalone HTML file.
